Academic Profile Website

Project Background

During my PhD I wanted to put together a small portfolio site for my talks, publications, etc. Some time later I was looking into Jekyll and I decided to refactor the site as a Jekyll project. It’s not really the best use-case for Jekyll, as I only update the information on it once every few months, but it was a fun exercise anyway, and now it’s set up it does make it a little quicker to update the site with new information.

Project Description

It’s a pretty simple static site, built with Jekyll, using my own CSS and minimal JavaScript to keep it as light as possible.

In terms of design I wanted it to be clean and minimal. I know from my own experience of having browsed lots of academics’ websites that 99% of the time you’re visiting the site looking for a specific piece of information about them or their work. So really you just want the key information to be easy to find at a glance.

To keep it responsive, I’ve built it around three different layouts: mobile, tablet, and laptop/desktop views. I’ve also tested it at each of the screen dimensions between the key breakpoints.

As it’s basically just text, I didn’t think it made sense to have a dedicated layout for extra large screens, so I just have the laptop layout capped at a maximum of 1200px. So if it’s viewed on larger screens than that, you’ll just get a centred version of the regular desktop version, to avoid stretching the text out onto very long lines.
